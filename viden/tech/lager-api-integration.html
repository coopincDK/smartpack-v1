Lager API-integration: hvad det er, hvornår du har brug for det og hvad det koster

9. maj 2026~7 min

Din webshop sender en ordre. WMS'et modtager den. Fragtportalen printer labelen. Alt sker automatisk. Det er hvad en velfungerende API-integration ser ud som.

Uden den: din medarbejder eksporterer en CSV fra webshoppen, importerer den i WMS'et, eksporterer igen til fragtportalen. 45 minutter om dagen ved 200 ordrer. 165 timer om året. 57.750 kr. i spildt løn ved 350 kr. i timen.

De tre integrationstyper på lager

REST API (realtidsintegration)

Systemerne taler direkte med hinanden. Ordre placeres i webshoppen kl. 14:32. WMS modtager den kl. 14:32. Beholdning opdateres kl. 14:33. Det er REST API.

Hvornår det giver mening: Over 100 daglige ordrer, multi-channel salg (webshop + marketplace + B2B), behov for realtids lagerstatus på hjemmesiden.

Hvad det koster: 5.000-25.000 kr. opsætning per integration, 0-500 kr./måned løbende. Teknisk kompetence krævet eller ekstern konsulent.

Webhooks (hændelsebaseret)

WMS'et sender en besked til webshoppen, når noget sker. Ordre er plukket, pakket, afsendt. Tracking-nummer genereret. Webshoppen opdaterer ordrestatus automatisk.

Hvornår det giver mening: Når du vil have automatiske statusopdateringer uden at polle API'et konstant. Lavere serverbelastning end fuld REST API.

Hvad det koster: Typisk billigere end fuld REST API. 2.000-10.000 kr. opsætning.

Fil-baseret (CSV/XML import/eksport)

Systemerne udveksler filer på et aftalt tidspunkt. Nattlig batch-kørsel: eksporter ordrer fra webshoppen, importer i WMS, eksporter bekræftelser. Gammelkendt, simpelt, fungerer.

Hvornår det giver mening: Under 50 daglige ordrer, simpel ét-kanal drift, lille teknisk budget, B2B-kunder der sender Excel-ordrer.

Hvad det koster: 0-5.000 kr. opsætning, men koster tid i manuel håndtering løbende.

De typiske integrationspunkter på et lager

De fleste lageroperationer har brug for integration på 3-4 punkter:

  • Webshop → WMS: Ordrer ind, beholdning ud
  • WMS → Fragtportal: Forsendelsesdata ind, tracking-nummer og label ud
  • WMS → ERP: Lagerværdi, lagerbevægelser, indkøbsordrer
  • Leverandør → WMS: Advanceret forsendelsesadvisering (ASN) ved varemodtagelse

De fleste begynder med webshop-integration og tilføjer fragtportal-integration. ERP-integration kommer typisk når virksomheden vokser til 500+ daglige ordrer.

Hvad der går galt uden ordentlig integration

Manuel dataoverførsel er ikke bare ineffektivt. Det er en fejlkilde.

  • Dobbelt data-entry: Ordrenummer tastes manuelt ind. Fejlrate på 1-3% per tastning. 200 ordrer om dagen = 2-6 fejl dagligt.
  • Forsinkede statusopdateringer: Kunden spørger til sin ordre kl. 15. Medarbejderen har ikke eksporteret tracking-numre endnu. De eksporteres kl. 16. En times unødvendig kundeservice.
  • Beholdning out of sync: Webshoppen viser 10 på lager. WMS'et ved at 3 er reserveret til en igangværende ordre. Du sælger 8 og kan kun levere 7.

Hvornår API er overkill

API-integration er ikke altid svaret. Det giver ikke mening at investere 20.000 kr. i en integration der sparer 30 minutter om måneden.

Fil-baseret integration er tilstrækkeligt hvis:

  • Under 30 daglige ordrer med én salgskanal
  • Ingen realtids lagerstatus på hjemmesiden
  • B2B-kunder accepterer daglig batch-opdatering
  • Ingen sæsonudsving der kræver hurtig skalering

Investér i API-integration når manuel overførsel tager mere end 30 minutter om dagen, eller når fejl fra manuel dataentry koster mere end 3.000 kr. om måneden i tid og fejlrettelse.

Ofte stillede spørgsmål

Hvad er forskellen på REST API og webhook?

REST API er en tovejsforbindelse du kan forespørge når du vil. Webhook er en envejsbesked systemet sender når noget sker. REST API bruges til at hente data. Webhooks bruges til at reagere på hændelser. De fleste integrationer bruger begge.

Hvad koster det at integrere WMS med webshop?

Standard Shopify eller WooCommerce integration koster 5.000-15.000 kr. i opsætning. Native-integrationer (som SmartPack) er typisk inkluderet i abonnementet. Custom EDI-integration til B2B-kunder koster 20.000-80.000 kr.

Har jeg brug for en teknisk medarbejder til at vedligeholde integrationen?

Standard cloud-integrationer vedligeholdes typisk af leverandøren og kræver ingen teknisk medarbejder. Custom-integrationer kræver enten intern teknisk kompetence eller en serviceaftale med leverandøren. Budget 5.000-20.000 kr./år til vedligeholdelse af custom-integrationer.

Hvornår er fil-baseret integration godt nok?

Under 50 daglige ordrer med én salgskanal og ingen krav til realtids lagerstatus. Fil-baseret integration er billigere og enklere. Skift til API når manuel overførsel tager mere end 30 minutter om dagen.
